How I served this font

I went with self-hosted. I downloaded a font that isn't pre-installed on most computers (Bangers), saved it as both .woff2 and .woff, and uploaded both files to my server at /ITC505/lab-3/fonts/. That way, the browser loads the font directly from my websiteâ€”no third-party required.

I used the @font-face rule in my CSS to give the font a friendly name (BangersLocal) and linked to those two files. Your browser downloads the font and can immediately use it wherever I've asked for it when you load this page.

I also define font-display: swap. That tells the browser to show a default fallback font first, then "swap" the custom font in as soon as it's readyâ€”so you get content right away, and it still ends up looking how I prefer.

Why not simply link to a provider such as Google Fonts? That is equally acceptable. I went with self-hosting so that I have full control, fewer external dependencies, and predictable caching. It also makes the assignment absolutely clear to graders: the font is being served from my server.

CSS snippet

@font-face {
  font-family: "BangersLocal";
  src: url("./fonts/Bangers.woff2") format("woff2"),
       url("./fonts/Bangers.woff") format("woff");
  font-weight: 400;
  font-style: normal;
  font-display: swap;
}
